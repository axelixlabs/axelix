# Профиль сервиса

Профиль сервиса — это раздел Axelix для одного сервиса. Он объединяет все экраны диагностики и проверки одного сервиса
Spring Boot за одной боковой панелью, чтобы можно было переключаться между бинами, логгерами, дампами потоков и
остальным, не теряя контекста рассматриваемого сервиса.

Чтобы войти в Профиль сервиса, нажмите на карточку сервиса на [Панели](./wallboard). Первый экран, на который вы
попадаете, — это [Подробности](../features/details).

## Структура

В Профиле сервиса есть два раздела:

- На **левой боковой панели** перечислены все страницы возможностей, доступные для сервиса.
- В **основной области контента** отображается текущая выбранная страница возможности.

## Группы боковой панели

Боковая панель разделена на три группы. Каждую группу можно развернуть или свернуть независимо, а текущая открытая
страница выделена.

### Аналитика

Общая информация о работе сервиса.

- [Подробности](../features/details): визитка сервиса. Показывает данные сборки, Git-ревизию, поставщика JDK и
  метаданные среды выполнения, плюс действие **Выгрузить состояние**, которое экспортирует ZIP-архив с выбранными
  диагностическими данными (дамп кучи, дамп потоков, логи, бины, кэши и т. д.) для оффлайн-анализа.
- [Метрики](../features/metrics): метрики Micrometer в реальном времени, предоставляемые сервисом, автоматически
  сгруппированные по префиксу в логические категории — JVM, HTTP, JDBC и т.д.. В каждую метрику можно провалиться, чтобы
  увидеть её текущее значение и параметры тегов.
- [Логгеры](../features/loggers/loggers.mdx): список всех настроенных логгеров и групп логгеров в сервисе с указанием
  текущего уровня. Уровень можно менять на лету (например, поднять пакет до `DEBUG`, пока воспроизводится баг) и
  сбросить обратно к исходному значению, когда закончите. [Также доступно как
  MCP-инструмент.](../setting-up-master-ui/mcp/mcp-tools.mdx#loggers)

### Spring Framework

Интроспекция, специфичная для Spring.

- [Окружение](../features/properties): разрешённый Spring `Environment` сервиса. Каждое активное свойство сгруппировано
  по источнику (системные свойства, `application.yaml`, аргументы командной строки и так далее), поэтому вы видите не
  только фактическое значение, но и то, какой источник свойств его предоставил. [Также доступно как
  MCP-инструмент.](../setting-up-master-ui/mcp/mcp-tools.mdx#instance-introspection)
- [Бины](../features/beans): каждый бин в контексте приложения с его областью видимости (Singleton, Prototype, Request,
  Session и другими), зависимостями, псевдонимами, квалификаторами и источником (`@Component`, `@Bean`-метод,
  `FactoryBean`, синтетический). Бины, за которыми стоит `@ConfigurationProperties`, ведут напрямую на соответствующую
  запись на странице Свойства конфигурации. [Также доступно как
  MCP-инструмент.](../setting-up-master-ui/mcp/mcp-tools.mdx#instance-introspection)
- [Свойства конфигурации](../features/configuration-properties): все бины, помеченные `@ConfigurationProperties`, и
  текущее значение каждого их поля. Позволяет увидеть, что в действительности содержится в ваших типизированных объектах
  конфигурации во время работы, вместо того чтобы выводить это из YAML. [Также доступно как
  MCP-инструмент.](../setting-up-master-ui/mcp/mcp-tools.mdx#instance-introspection)
- [Запланированные задачи](../features/scheduled-tasks): каждая `@Scheduled` задача в сервисе, сгруппированная по типу
  триггера (**cron**, **фиксированная задержка**, **фиксированная частота**). Задачу можно приостановить или
  возобновить, отредактировать её расписание и запустить разовый прогон вручную. [Также доступно как
  MCP-инструмент.](../setting-up-master-ui/mcp/mcp-tools.mdx#instance-introspection)
- [Условия](../features/conditions): результат вычисления Spring-условий `@Conditional` для каждого бина-кандидата,
  разделённый на **положительные** и **отрицательные** совпадения. Эта страница — самый быстрый способ выяснить, какое
  условие помешало автоконфигурации поставить ожидаемый вами бин. [Также доступно как
  MCP-инструмент.](../setting-up-master-ui/mcp/mcp-tools.mdx#instance-introspection)
- [Кэши](../features/caches): Spring-кэши, зарегистрированные в сервисе, сгруппированные по их кэш менеджерам.
  Показывает частоту попаданий и промахов, примерное количество записей и позволяет очищать отдельные кэши или сразу
  все. [Также доступно как MCP-инструмент.](../setting-up-master-ui/mcp/mcp-tools.mdx#caches)
- [Мониторинг транзакций](../features/transaction-control): представление каждой транзакции, которую выполняет сервис, в
  реальном времени — с исходным классом, методом и метриками длительности. Каждое выполнение открывает временную шкалу
  SQL-запросов, прошедших внутри транзакции, — вы видите точный SQL, его время начала и окончания и продолжительность, —
  это и есть способ замечать паттерны доступа N+1 и другие неэффективности на уровне запросов.

### JVM

Диагностика на уровне JVM.

- [Дамп потоков](../features/thread-dump): снимок каждого живого потока JVM с его состоянием, стеком вызовов, счётчиками
  blocked/waited и информацией о владельце блокировки. На странице также можно переключать мониторинг конкуренции
  потоков за блокировки на работающей JVM.
- [Сборщик мусора](../features/garbage-collector): логи сборщика мусора, поступающие из сервиса. Логирование сборщика
  мусора обычно отключено в продакшен-JVM по умолчанию, поэтому страница выставляет действие **Включить логирование
  сборщика мусора**, которое его включает без перезапуска сервиса.

## Доступ через MCP

Некоторые из перечисленных выше функций также доступны как инструменты MCP-сервера, встроенного в Axelix Master, так что
ИИ-агент, подключённый по MCP, может прочитать те же данные. Полный каталог зарегистрированных инструментов можно
посмотреть на [экране MCP](./mcp.mdx), а актуальный справочный список с именами, тегами и разграничением доступа по
ролям живёт в [MCP-инструменты](../setting-up-master-ui/mcp/mcp-tools.mdx).
