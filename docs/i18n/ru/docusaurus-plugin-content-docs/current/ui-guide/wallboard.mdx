# Панель

Панель — лендинг-страница по умолчанию в Axelix. Она показывает каждый Spring Boot-сервис, обнаруженный Axelix Master, в
виде сетки компактных карточек — у каждой свой текущий статус здоровья и краткая сводка того, что на ней работает. Клик
по карточке открывает [Профиль сервиса](./service-profile).

## Карточки сервисов

Каждая карточка соответствует одному обнаруженному сервису. В шапке карточки — имя сервиса и индикатор статуса справа:

- **Зелёная** точка — сервис **UP**.
- Пульсирующая **красная** точка — сервис **DOWN**.
- Иконка **вопроса** — Axelix Master не смог определить статус сервиса (**UNKNOWN**).

Карточки не в состоянии **UP** отрисовываются с фоном цвета статуса, чтобы выделяться в сетке.

В теле карточки перечислены детали рантайма, которые Axelix собрал при регистрации сервиса:

- **Version**: версия самого сервиса (версия Maven/Gradle-артефакта).
- **Spring Boot**: версия Spring Boot, под которую собран сервис.
- **Java**: мажорная версия Java у работающей JVM.
- **Commit**: короткий Git SHA сборки (из метаданных в стиле `git-commit-id`, если приложение их отдаёт).
- **Deployed for**: сколько времени сервис зарегистрирован в Master.

Клик в любом месте карточки ведёт на экран **Детали** [Профиля сервиса](./service-profile).

## Поиск и фильтры

Поле поиска над сеткой фильтрует карточки по имени сервиса (поиск подстроки без учёта регистра). Счётчик справа
показывает `<совпадений> / <всего>`.

Кнопка **+ Добавить фильтр** рядом с поиском открывает всплывающее окно с тремя полями:

- **Поле** — одно из **Spring Boot**, **Java**, **Spring Framework**, **Kotlin**.
- **Сравнение** — **Равно**, **Меньше или равно** или **Больше или равно**. Сравнение учитывает semver, поэтому `<= 3.2`
  совпадёт с `3.1.x` и `3.2.x`, но не с `3.3.x`.
- **Значение** — версия, взятая из сервисов, которые сейчас в парке. Предлагаются только те версии, которые встречаются
  хотя бы на одной карточке, поэтому собрать фильтр, ничего не находящий, нельзя.

При сохранении фильтр добавляется как удаляемый тег между поиском и сеткой, сужает список показываемых карточек и
записывается в URL как параметр `?f=Field:Operator:Value`. Отфильтрованные URL Панели можно сохранить в закладки и
поделиться ими, а [Дашборд](./dashboard) сам генерирует их при клике по сектору диаграммы «Карта системы». Добавляйте
новые фильтры повторным действием — для отображения карточки должны совпасть все активные фильтры.

## Доступ через MCP

Данные Панели доступны как инструмент MCP — ИИ-агент, подключённый к встроенному MCP-серверу Axelix Master, может
перечислить каждый управляемый сервис с теми же полями, что показаны на карточках. [См.
MCP-каталог.](../setting-up-master-ui/mcp/mcp-tools#instances)
