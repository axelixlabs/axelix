# Дашборд

Дашборд — лендинг-страница в Axelix. На ней представлена информация обо всех сервисах Spring Boot, которые в настоящее
время отслеживает Axelix Master: версии в парке сервисов, количество работоспособных сервисов в данный момент и общий
объем используемой памяти.

Данные собираются Axelix Master из каждого приложения Spring Boot, в котором используется стартер Axelix. До регистрации
первого сервиса страница отображается пустой.

## Карта системы

**Карта системы** — это ряд круговых диаграмм в верхней части страницы. Каждая диаграмма разбивает парк сервисов по
одному программному компоненту:

- **Spring Boot**: распределение версий Spring Boot среди всех обнаруженных сервисов.
- **Spring Framework**: распределение версий Spring Framework.
- **Java**: распределение мажорных версий Java (например, `17`, `21`).
- **Kotlin**: распределение версий Kotlin. Чисто Java-сервисы сюда ничего не вносят, поэтому в парке без Kotlin-сервисов
  эта диаграмма пустая.

В подписях к срезам указано количество сервисов и их доля в формате value (percent%). Например, 12 (60%) означает, что
12 сервисов используют эту версию и составляют 60 % компонента диаграммы.

### Клик по сектору открывает Панель

Клик по сектору ведёт на [Панель](./wallboard) с уже применённым фильтром по компоненту и версии, на которые вы
кликнули. Все четыре перечисленные выше диаграммы кликабельны.

## Состояние работоспособности

**Состояние работоспособности** — кольцевая диаграмма, показывающая текущий статус каждого обнаруженного сервиса.
Отслеживаются три статуса:

- **UP** (зелёный): сервис здоров.
- **DOWN** (красный): сервис нездоров.
- **UNKNOWN** (серый): Axelix Master не смог определить статус сервиса.

Число в центре кольца подписано как **Общее количество** и равно сумме всех трёх статусов, то есть общему числу
сервисов, которые сейчас отслеживает Axelix.

## Статистика

**Статистика** — это ряд из трех карточек рядом с графиком состояния здоровья:

- **Общее количество сервисов**: то же общее число, что показано в центре кольца «Состояние работоспособности».
- **Средний размер кучи**: средний размер heap JVM среди всех обнаруженных сервисов.
- **Общий размер кучи**: суммарный размер heap JVM среди всех обнаруженных сервисов.

Оба значения heap перед выводом приводятся к удобочитаемой единице измерения, поэтому в карточке может быть `512 MB`, `2
GB` и т.п. — в зависимости от размера парка.
