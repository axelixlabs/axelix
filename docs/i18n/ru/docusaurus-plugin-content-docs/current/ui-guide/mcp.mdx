# MCP

На экране **MCP** перечислены все инструменты, которые Axelix Master предоставляет через встроенный сервер Model Context
Protocol. Каждая карточка — это отдельный инструмент, который ИИ-агент, подключённый по MCP, может вызвать в отношении
любого зарегистрированного сервиса Spring Boot.

## Когда появляется вкладка MCP

Ссылка **MCP** на панели навигации отображается, только когда MCP-сервер включён в Master. Управляет этим свойство
`axelix.master.mcp-server.enabled`, по умолчанию имеющее значение `true`. В развёртываниях, где сервер отключён, ссылка
скрыта, а `/mcp-server` недоступен. Подробности о свойстве см. в разделе [Настройка
MCP-сервера](../setting-up-master-ui/configuring-master/configuring-master.mdx#mcp-сервер).

Сервер MCP выполняет аутентификацию агентов ИИ отдельно от сеанса работы в браузере. Учётные данные, поддерживаемые
схемы `Basic` и `Bearer`, а также разграничение полномочий по каждому инструменту описаны в разделе [Аутентификация на
MCP-сервере](../setting-up-master-ui/authentication/authentication.mdx#аутентификация-на-mcp-сервере). Эта страница
описывает только интерфейс каталога, доступный только для чтения, она не влияет на то, какие инструменты может вызывать
агент.

## Каталог инструментов

Каталог представляет собой сетку карточек. Поле поиска вверху фильтрует карточки по названию, а счётчик справа
показывает `<совпадения> / <всего>`. Поиск ищет совпадение в любой части названия инструмента и работает без учёта
регистра.

Каждая карточка состоит из трёх одинаковых частей:

- **Заголовок** с названием инструмента и цветной точкой состояния — зелёной, когда инструмент зарегистрирован на
  MCP-сервере, и красной, когда он в данный момент недоступен.
- **Описание** — краткая аннотация, которую инструмент предоставляет MCP-клиентам. Длинные описания обрезаются, чтобы
  увидеть полный текст, наведите курсор на описание.
- **Нижний колонтитул** с тегами аннотаций — см. [Аннотации инструмента](#tool-annotations) ниже.

## Аннотации инструмента

Каждый инструмент объявляет четыре логические подсказки, которые сообщают MCP-клиентам, каких побочных эффектов следует
ожидать при его вызове. Две из них всегда видны прямо в нижнем колонтитуле карточки, а все четыре раскрываются через
всплывающее окно.

**Встроенные теги** (всегда видны):

- **Только для чтения** — инструмент не изменяет состояние целевого сервиса Spring Boot.
- **Идемпотентный** — многократный вызов инструмента с одними и теми же аргументами даёт тот же эффект, что и
  однократный вызов.

Каждый тег окрашен в зелёный, когда подсказка <code style={{color: 'green'}}>true</code>, и в оранжевый, когда она <code
style={{color: 'orange'}}>false</code>. Отображаемое значение — это собственное утверждение инструмента, а не гарантия
того, что на самом деле делает базовый эндпоинт.

**Всплывающее окно с полными аннотациями** — нажмите на иконку меню (≡) справа в нижнем колонтитуле, чтобы открыть
панель со всеми четырьмя подсказками и их логическими значениями:

- **Только для чтения** — инструмент не изменяет состояние целевого сервиса Spring Boot.
- **Идемпотентный** — многократный вызов инструмента с одними и теми же аргументами даёт тот же эффект, что и
  однократный вызов.
- **Разрушительный** — инструмент может удалить данные или иным образом внести изменения, которые нельзя отменить из
  интерфейса.
- **Открытый мир** — инструмент обращается за пределы Master и зарегистрированных сервисов Spring Boot (например, делает
  внешний HTTP-вызов).

## Чего на этом экране нет

Страница MCP — это место для обзора инструментов, а не для их запуска. Отсюда нельзя:

- Вызвать инструмент для работы с сервисом Spring Boot — это делается из вашего MCP-клиента (Claude Desktop, Cursor,
  кастомный агент и т. д.).
- Включить или отключить отдельные инструменты — в каталоге отображается то, что Master зарегистрировал при запуске.
- Просмотреть историю вызовов или метрики по каждому инструменту.

Чтобы воспользоваться каталогом, укажите в MCP-клиенте адрес `<master-base-url>/api/mcp` и пройдите аутентификацию, как
описано в разделе [Аутентификация на
MCP-сервере](../setting-up-master-ui/authentication/authentication.mdx#аутентификация-на-mcp-сервере).

:::info
**Журнал доступа** для вызовов MCP-инструментов есть в плане разработки и будет поставлен вместе с Axelix Enterprise. И
журнал доступа, и сам Axelix Enterprise всё ещё в разработке, поэтому пока экран MCP остаётся каталогом только для
чтения без истории и метрик по каждому инструменту.
:::
