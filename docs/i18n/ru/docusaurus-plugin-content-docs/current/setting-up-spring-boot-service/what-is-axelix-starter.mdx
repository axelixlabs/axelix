# Что такое Axelix Starter?

Axelix Starter (также известный как ASBS — Axelix Spring Boot Starter) — это Spring Boot starter-зависимость, которая
позволяет вашим Spring Boot-сервисам бесшовно интегрироваться с платформой Axelix. Это мост между вашими сервисами и
консолью управления Axelix Master.

## Обзор

Axelix Starter превращает ваш обычный сервис на Spring Boot в «управляемый сервис», который можно мониторить, отлаживать
и контролировать через интерфейс Axelix Master. Он работает за счёт расширения нативных возможностей Spring Boot и
предоставляет дополнительные эндпоинты и функциональность.

## Ключевые возможности

### **Автоматическое обнаружение и регистрация сервисов**
- **Автоматическое обнаружение**: Axelix Master автоматически регистрирует сервисы
- **Поддержка нескольких сред**: работает в Kubernetes, Docker Compose
- **Самовосстанавливающаяся регистрация**: автоматическая повторная регистрация, если соединение прервалось

### **Расширенный мониторинг и отладка**
- **Анализ бинов в реальном времени**: просмотр всех Spring-бинов.
- **Инспекция свойств конфигурации**: значения свойств и их источники
- **Метрики времени выполнения**: метрики производительности и состояния сервиса
- **Экспорт состояния**: экспорт состояния сервиса (включая heap-dump, thread-dump, env и т. д.)

## Когда использовать Axelix Starter

### **Разработка и тестирование**
- **Отладка проблем конфигурации** в различных средах (локальной, промежуточной, рабочей)
- **Понимание связывания бинов и зависимостей** без глубокого погружения в код
- **Тестирование изменений конфигурации** без переразвёртывания или перезапусков
- **Валидация разрешения свойств** в нескольких источниках конфигурации
- **Переключение запланированных задач**: тестирование разных сценариев планирования с включением/выключением
  `@Scheduled`-методов
- **Управление кэшами**: проверка поведения кэшей очисткой или отключением кэшей во время тестирования

### **Продакшен-операции**
- **Мониторинг состояния сервиса** в разных средах и на разных сервисах
- **Экстренная отладка** во время продакшен-инцидентов
- **Управление конфигурацией** на нескольких сервисах
- **Аудит состояния и конфигурации сервиса** для устранения неполадок
