# Что такое Master?

Axelix Master (или просто «Master») — это центральный хаб экосистемы Axelix, сервис с расширенными возможностями для
управления и мониторинга сервисов на Spring Boot в различных средах.

## Основная задача

Axelix Master выступает в роли **централизованной консоли управления**, которая автоматически обнаруживает, регистрирует
и мониторит сервисы на Spring Boot, обеспечивая инсайты и контроль в реальном времени.

## Ключевые возможности

### 🔍 **Автоматическое обнаружение сервисов**
Master автоматически находит и регистрирует сервисы на Spring Boot в вашей среде:

**Поддерживаемые среды:**
- ✅ **Kubernetes**
- ✅ **Docker Compose**
- 🔄 **Скоро будут добавлены другие среды**

**Как работает обнаружение:**
1. **Определение среды**: Master определяет runtime-среду
2. **Сканирование сервисов**: ищет сервисы с подключённым Axelix SBS
3. **Автоматическая регистрация**: регистрирует подходящие сервисы автоматически
4. **Мониторинг состояния**: периодически опрашивает зарегистрированные сервисы на предмет статуса

### 🖥️ **Единый веб-интерфейс**
Единое окно для всех ваших сервисов на Spring Boot:

**Включает в себя:**
- Дашборд со всеми зарегистрированными сервисами
- Состояние и метрики в реальном времени
- Инспекция свойств конфигурации и окружения
- Детальный анализ бинов
- и так далее, см. раздел «Возможности»

### 🔄 **Прокси и оркестрация сервисов**
Master выступает в роли умного прокси между пользователями и управляемыми сервисами.
