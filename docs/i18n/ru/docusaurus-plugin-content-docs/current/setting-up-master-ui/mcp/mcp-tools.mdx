# MCP-инструменты

На этой странице представлен список всех инструментов, доступных агентам искусственного интеллекта на сервере MCP в
Axelix Master. В каждой записи указано название инструмента, теги аннотаций, которые он поддерживает, состояние
активации, роли, которым разрешён доступ к инструменту, а также краткое описание его функций. Тот же каталог доступен в
браузере на [экране MCP](../../ui-guide/mcp). Эта страница является основным источником информации для операторов,
которые планируют, кому и что можно разрешить.

В списке предполагается, что сервер MCP включён (`axelix.master.mcp-server.enabled=true`, по умолчанию). Если для этого
свойства установлено значение `false`, сервер вообще не регистрируется, ни один из перечисленных ниже инструментов
недоступен, а ссылка **MCP** скрыта в пользовательском интерфейсе. См. [Конфигурация сервера
MCP](../configuring-master/configuring-master.mdx#mcp-сервер).

## Состояние инструмента

Все перечисленные ниже инструменты входят в состав Master и регистрируются на сервере MCP при запуске. Для каждого
инструмента нет переключателя, позволяющего включать или отключать его, — каталог фиксирован в сборке. Инструмент
становится недоступным только в том случае, если отключён сам сервер MCP или не запущен основной процесс. В [каталоге
пользовательского интерфейса](../../ui-guide/mcp#the-tool-catalog) точка состояния на каждой карточке отражает текущее
состояние регистрации: зелёная — если сервер зарегистрировал инструмент, красная — если в данный момент он недоступен.

## Аннотационные теги инструмента

Каждый инструмент объявляет четыре логических аннотационных тега, которые сообщают MCP-клиентам, каких побочных эффектов
следует ожидать. Теги ниже — это значения, которые объявляют сами инструменты Axelix, это подсказки для вызывающего
агента, а не проверки авторизации, которые применяет Master.

- **только для чтения** — инструмент не изменяет состояние целевого сервиса Spring Boot.
- **идемпотентный** — многократный вызов инструмента с одними и теми же аргументами даёт тот же эффект, что и
  однократный вызов.
- **разрушительный** — инструмент может удалить данные или внести изменения, которые нельзя отменить из интерфейса.
- **открытый мир** — инструмент обращается за пределы Master и зарегистрированных сервисов Spring Boot.

## Доступ по ролям

MCP-фильтр (`McpAuthorizationFilter`) аутентифицирует агента так же, как интерфейс аутентифицирует сеанс в браузере.
Схемы учётных данных см. в разделе [Аутентификация на
MCP-сервере](../authentication/authentication.mdx#аутентификация-на-mcp-сервере).

### Сервисы

Единственный инструмент в этой группе возвращает полный реестр Master по управляемым сервисам Spring Boot. Это отправная
точка для любой задачи, начинающейся с «найди сервис по имени и получи его `instanceId`» — `instanceId` нужен каждому
другому инструменту.

### Инспекция сервиса

Представления только для чтения по одному сервису Spring Boot: его Spring-бины, свойства окружения, бины
`@ConfigurationProperties`, отчёт об автоконфигурации и зарегистрированные запланированные задачи. Каждый инструмент в
этой группе принимает `instanceId`, полученный через `getWallboard`.

См. также: [Бины](../../features/beans), [Окружение](../../features/properties), [Свойства
конфигурации](../../features/configuration-properties), [Условия](../../features/conditions), [Запланированные
задачи](../../features/scheduled-tasks), [Профиль сервиса — UI-гайд](../../ui-guide/service-profile).

всех кэш менеджеров сразу, либо в конкретном именованном кэше внутри выбранного кэш менеджера.
