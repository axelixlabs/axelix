# Метрики

На странице **Метрики** перечислены все метрики Micrometer, предоставляемые управляемым сервисом Spring Boot, с
динамической диаграммой и фильтрацией по тегам для каждой метрики.

Метрики автоматически группируются по префиксам в раскрывающиеся выпадающие списки. Каждая группа представляет собой
логическую категорию связанных метрик, таких как JVM, HTTP, JDBC и т.д.. Такая организация помогает быстро находить и
анализировать метрики, относящиеся к одной функциональной области.

![главная страница метрик](../../../../../static/img/feature/metrics/metrics-main-page.png) ***Метрики в интерфейсе
Axelix***

Страница доступна только для чтения всем пользователям, прошедшим аутентификацию: ее могут открыть `VIEWER`, `EDITOR`,
`ADMIN` и `SUPER_ADMIN` Полную матрицу ролей и полномочий см. в разделе [Роли и
полномочия](../setting-up-master-ui/authentication/authentication.mdx#roles-and-authorities).

## Метрики, публикуемые Axelix

Помимо метрик, которые уже экспортирует само приложение, Axelix Spring Boot Starter может добавлять и собственные метрики Micrometer. Это происходит,
 если в приложении уже есть бин `MeterRegistry`.

- `axelix_transaction_duration` — `Timer` с тегами `class`, `method` и `status=success|error`.
- `axelix_transaction_queries` — `Counter` с тегами `class` и `method`.
- `axelix_cache_requests` — `Counter` с тегами `cache` и `result=hit|miss`.
- `axelix_cache_enabled` — `Gauge` с тегом `cache`, который возвращает `1` для включённых кэшей и `0` для выключенных.

Эндпоинт `/actuator/prometheus` также будет экспортировать эти метриаи по-умолчанию (если prometheus эндпоинт, конечно, включён). Это означает, что
впоследствие вы сможете использовать данные метрики для построения своих собственных дашбордов, например, в Grafana.

Больше деталей доступно [в данной секции документации](../installation/configuring-spring-boot-starter#export-axelix-metrics-through-micrometer).

## Структура страницы

### Заголовок выпадающего списка
- **Имя метрики**: полный идентификатор метрики (например, `jvm.memory.max`)
- **Описание**: наведите курсор на иконку ![иконка
  подсказки](../../../../../static/img/feature/icons/question-icon.png), чтобы увидеть, что измеряет эта метрика.

### Левая панель: информация о текущей метрике
Показывает информацию о выбранной метрике в реальном времени:
- **Значение**: последнее измерение
- **Базовая Единица**: единица измерения (байты, миллисекунды, проценты, скалярная величина и т. д.)
- **Теги**: раскрывающийся список для фильтрации измерений по параметрам тегов. Предлагаются только те комбинации,
  которые действительно существуют на сервисе, недопустимые комбинации отображаются, но не используются.


### Правая панель: график временного ряда
Линейный график изменения значения метрики в скользящем временном окне. Страница периодически опрашивает сервис и
сбрасывает окно по истечении заданного времени, поэтому график отражает текущее состояние, а не всю историю с момента
запуска сервиса.

## Фильтрация по тегам (детализация)

Список допустимых комбинаций тегов берётся из Micrometer `MeterRegistry` сервиса — Axelix не придумывает и не
ограничивает их количество. Когда вы выбираете значение в одном выпадающем списке, в остальных отображаются только
допустимые варианты. Некорректные варианты остаются видимыми, но неактивными, с подсказкой «Это значение тега
недопустимо, учитывая значения, выбранные для других тегов». Подробнее об этом механизме см. в документации по Spring
Boot: [Drilling Down](https://docs.spring.io/spring-boot/api/rest/actuator/metrics.html).

### Как это работает
Когда вы выбираете значение тега, доступные варианты для остальных тегов фильтруются до тех комбинаций, которые
действительно существуют в работающем сервисе.

### Пример: метрики памяти JVM
Для метрики `jvm.memory.used` с тегами `area` и `id`:

| `area`  | `id`                          | Доступно |
|---------|-------------------------------|----------|
| heap    | G1 Eden Space                 | ✅        |
| heap    | G1 Survivor Space             | ✅        |
| heap    | G1 Old Gen                    | ✅        |
| nonheap | Metaspace                     | ✅        |
| nonheap | Compressed Class Space        | ✅        |
| nonheap | CodeHeap profiled nmethods    | ✅        |
| nonheap | CodeHeap non-nmethods         | ✅        |
| nonheap | CodeHeap nonprofiled-nmethods | ✅        |
| heap    | Metaspace                     | ❌        |
| nonheap | G1 Old Gen                    | ❌        |

## Свойства

Страница работает поверх actuator-эндпоинта `axelix-metrics`, который добавляет Axelix Spring Boot Starter. Она доступна
через стандартные свойства Spring Boot Actuator. Полный список конечных точек Axelix и сопутствующих настроек см. в
разделе [Настройка Spring Boot
Starter](../setting-up-spring-boot-service/configuring-axelix-starter/configuring-axelix-starter.mdx):

<Tabs groupId="spring-config">
  <TabItem value="properties" label="application.properties">

```properties
management.endpoints.web.exposure.include=axelix-metrics
```

  </TabItem>
  <TabItem value="yaml" label="application.yaml">

```yaml
management:
  endpoints:
    web:
      exposure:
        include:
          - axelix-metrics
```

  </TabItem>
</Tabs>
