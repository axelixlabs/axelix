# Бины

На странице **Бины** перечислены все бины Spring, зарегистрированные в `ApplicationContext` управляемого сервиса. Каждая
строка представляет собой раскрывающийся список, в котором указаны класс бина, область видимости, зависимости,
квалификаторы, стратегия проксирования и исходный компонент Spring, использованный для его создания.

![главная страница бинов](../../../../../static/img/feature/beans/beans-main-page.png) ***Страница «Бины» в интерфейсе
Axelix***

Страница доступна только для чтения всем пользователям, прошедшим аутентификацию: ее могут открыть`VIEWER`, `EDITOR`,
`ADMIN` и `SUPER_ADMIN`. Полную матрицу ролей и полномочий см. в разделе [Роли и
полномочия](../setting-up-master-ui/authentication/authentication.mdx#roles-and-authorities).

Прокручиваемый список всех бинов с полем поиска над списком и счётчиком вида `<соответствует> / <всего>`.

В заголовке каждой строки рядом отображаются три элемента:

- **Имя бина** — идентификатор бина в Spring-контексте.
- **Имя класса** — полное имя класса (например, `org.springframework.scheduling.concurrent.ThreadPoolTaskExecutor`).
- **Область видимости** — `Tag` справа в заголовке, с цветовой кодировкой. Axelix сопоставляет строку области видимости,
  о которой сообщает Spring, с одним из следующих цветов:
    - <span style={{color: '#1677ff', fontWeight: 'bold'}}>singleton</span>
    - <span style={{color: '#fa8c16', fontWeight: 'bold'}}>prototype</span>
    - <span style={{color: '#13c2c2', fontWeight: 'bold'}}>request</span>
    - <span style={{color: '#a0d911', fontWeight: 'bold'}}>session</span>
    - <span style={{color: '#faad14', fontWeight: 'bold'}}>application</span>
    - <span style={{color: '#722ed1', fontWeight: 'bold'}}>websocket</span>
    - <span style={{color: '#fa541c', fontWeight: 'bold'}}>refresh</span> (Spring Cloud)
    - <span style={{color: '#eb2f96', fontWeight: 'bold'}}>пользовательские scope</span> (пользовательские области
      видимости)

## Раскрывающаяся панель подробностей бина

При раскрытии строки отображаются сведения о компоненте. Разделы отображаются только при наличии данных — например, для
компонента без квалификаторов строка «Квалификаторы» отображаться не будет.

![страница подробностей бина](../../../../../static/img/feature/beans/beans-details-page.png) ***Выпадающий блок бина в
интерфейсе Axelix***

- **Зависимости** — имена других бинов, от которых зависит этот бин. Если зависимость сама зарегистрирована как бин,
  клик по её имени переходит к этому бину в списке.
- **Алиасы** — альтернативные имена, зарегистрированные для этого бина, если такие есть.
- **Квалифаеры** — значения `@Qualifier`, применённые к бину, если такие есть.
- **Проксирование** — способ, которым Spring обернул сам бин. Один из следующих вариантов:
    - **CGLIB** — прокси на основе подкласса CGLIB.
    - **JDK Динамический Прокси** — прокси на основе интерфейса JDK.
    - **Не проксирован** — бин используется как есть.
    - **Неизвестно** — Axelix не смог определить тип прокси по определению бина.
- **Ленивый** — логическое значение `true` для компонентов, создаваемых при первом использовании, а не при запуске.
- **Primary** — логическое значение `true` для бинов, помеченных аннотацией `@Primary`.
- **Происхождение** — способ создания компонента в Spring. См. ниже раздел [Происхождение бина](#происхождение-бина).

### Происхождение бина

Раздел «Происхождение» показывает либо одну метку, либо небольшое раскрываемое дерево — в зависимости от того, какое
происхождение Spring сообщает для бина:

- **`@Component` или его варианты** — бин получен из `@Component` / `@Service` / `@Repository` / `@Controller` /
  пользовательского стереотипа. Если объявляющий класс отслеживается на странице [Условия](./conditions), метка меняется
  на **`Класс Авто-Конфигурации`** и становится ссылкой на эту страницу.
- **`@Bean` метод** — бин получен из фабричного метода `@Bean`. Дерево показывает:
    - имя **`@Bean` метода**.
    - **Объявляющий класс** — класс, в котором определён `@Bean` метод, доступный для перехода к компоненту, класс
      которого соответствует, когда он зарегистрирован. Метка меняется на **«Класс Авто-Конфигурации»**, когда
      объявляющий класс отслеживается на странице [Условия](./conditions).
    - **Кто определяет этот компонент?** ссылка на страницу [Условия](./conditions) — показывается в том же случае, т.
      е. когда объявляющий класс там отслеживается.
    - `@ConfigurationProperties` бин - ссылка на страницу [Configuration Properties](./configuration-properties) (только
      если бин помечен аннотацией `@ConfigurationProperties`).
- **Создан через `FactoryBean`** — бин был создан `FactoryBean`. Дерево показывает имя класса **Factory Bean**.
- **Синтетический бин** — внутренний бин Spring (например, инфраструктурные объекты, регистрируемые фреймворком или
  библиотеками вроде Spring Data).
- **Неизвестно** — Spring не сообщил достаточно метаданных, чтобы классифицировать происхождение. Для бинов,
  зарегистрированных программно через `BeanDefinitionRegistry`, это обычный результат. Если такой бин также помечен
  аннотацией `@ConfigurationProperties`, метка «Происхождение» переключается на **бин `@ConfigurationProperties`** и
  ссылается на страницу [Configuration Properties](./configuration-properties).

## Свойства

Страница работает поверх actuator-эндпоинта `axelix-beans`, который добавляет Axelix Spring Boot Starter. Она доступна
через стандартные свойства Spring Boot Actuator. Полный список конечных точек Axelix и сопутствующих настроек см. в
разделе [Настройка Spring Boot
Starter](../setting-up-spring-boot-service/configuring-axelix-starter/configuring-axelix-starter.mdx):

<Tabs groupId="spring-config">
  <TabItem value="properties" label="application.properties">

```properties
management.endpoints.web.exposure.include=axelix-beans
```

  </TabItem>
  <TabItem value="yaml" label="application.yaml">

```yaml
management:
  endpoints:
    web:
      exposure:
        include:
          - axelix-beans
```

  </TabItem>
</Tabs>
