# Логгеры

Страница **Логгеры** даёт доступ ко всем настроенным механизмам логирования в сервисе на Spring Boot, группам логгеров и
позволяет менять их уровни логирования.

![главная страница логгеров](../../../../../../static/img/feature/loggers/loggers-main-page.png) ***Страница «Логгеры» в
интерфейсе Axelix***

Страница доступна для всех пользователей, прошедших аутентификацию:`VIEWER`, `EDITOR`, `ADMIN` и `SUPER_ADMIN` могут
открывать ее, изменять уровни доступа и сбрасывать их. Полную матрицу ролей и полномочий см. в разделе [Роли и
полномочия](../../setting-up-master-ui/authentication/authentication.mdx#roles-and-authorities).

## Вкладка «Логгеры»
Прокручиваемый список со всеми активными логгерами сервиса, с индикатором их уровней логирования, функцией поиска для
удобной навигации и счётчиком активных логгеров вида `<соответствует> / <всего>`.

- **Имя**:                  имя логгера.
- **Уровень логирования**:  текущий уровень логгера. (См. **Интерактивные возможности**)
- **Сбросить**:             сбрасывает уровень логирования к начальному значению. (См. **Интерактивные возможности**)

## Вкладка «Группы логгеров»
![главная страница групп логгеров](../../../../../../static/img/feature/loggers/loggers-main-group-page.png) ***Страница
«Группы логгеров» в интерфейсе Axelix***

Прокручиваемый список со всеми группами логгеров, с индикатором их уровней логирования, поиском по группам и логгерам
для удобной навигации и счётчиком групп.

- **Имя**:                    имя группы логгеров.
- **Уровень группы логгеров**: текущий уровень логирования группы. Если ни один уровень не выделен цветовым индикатором,
  значит у логгеров внутри группы разные уровни логирования. (См. **Интерактивные возможности**)

## Интерактивные возможности

Мы предоставляем возможность менять уровень логирования как отдельного логгера, так и группы логгеров.

### Логгеры
Например, у логгера `com.axelixlabs.axelix.sbs` уровень логирования <INFO /> (стартовое значение). Чтобы сменить его
уровень, нажмите на нужный уровень, например <WARN />. Логгер обновится, и выбранный уровень будет помечен иконкой
<TargetThemeIcon />. Учтите, что при смене уровня логирования `com.axelixlabs.axelix` с <INFO /> на <DEBUG /> (шаг 2) он
также будет помечен иконкой <TargetThemeIcon />, а все логгеры с префиксом `com.axelixlabs.axelix` изменят свой уровень
на <DEBUG /> (шаг 2). Однако они не будут помечены иконкой <TargetThemeIcon />, так как их уровень совпадает с
родительским логгером `com.axelixlabs.axelix`.

<LoggersTable />

Также мы предоставляем возможность сбросить уровень логирования для конкретного логгера:
1. Изначально у логгера `com.axelixlabs.axelix.sbs` был уровень <TRACE /> (стартовое значение).
2. Затем он был сменён на уровень логирования <INFO /> (шаг 1).
3. Кнопкой **Сбросить** можно вернуть логгеру его исходный уровень <TRACE /> (шаг 2).

<LoggersTableReset />

### Группы логгеров
- Если у логгеров в группе разные уровни, у группы нет цветового индикатора в строке уровня.
- Изменение уровня, например, у группы `web` на <WARN /> применяет <WARN /> ко всем логгерам этой группы.
- Сброс уровня группы логгеров не поддерживается — сбрасывать можно только отдельные логгеры.

## MCP-инструменты

Чтение дерева логгеров и изменение уровней во время работы — для отдельного логгера, группы логгеров или сброса
отдельного логгера — также доступны ИИ-агентам через MCP. См. [Каталог
MCP-инструментов](../../setting-up-master-ui/mcp/mcp-tools.mdx#loggers).

## Свойства

Страница работает поверх actuator-эндпоинта `axelix-loggers`, который добавляет Axelix Spring Boot Starter. Она доступна
через стандартные свойства Spring Boot Actuator. Полный список конечных точек Axelix и сопутствующих настроек см. в
разделе [Настройка Spring Boot
Starter](../../setting-up-spring-boot-service/configuring-axelix-starter/configuring-axelix-starter.mdx):

<Tabs groupId="spring-config">
  <TabItem value="properties" label="application.properties">

```properties
management.endpoints.web.exposure.include=axelix-loggers
```

  </TabItem>
  <TabItem value="yaml" label="application.yaml">

```yaml
management:
  endpoints:
    web:
      exposure:
        include:
          - axelix-loggers
```

  </TabItem>
</Tabs>
